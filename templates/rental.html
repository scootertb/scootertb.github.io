Project Overview

Link to the project repository

This is a project that incorporates a Python Flask app with HTML rendering and a SQL database to create a dashboard for keeping track of rental properties. This dashboard is capable of data submission, data viewing, and report generation. The reports that are generated gives the property owner insight into total income, total expenses, tenant balance, and an item-by-item breakdown on a per unit basis.

Certain aspects of this project that I want to highlight that made me deepen my understanding of working with SQL as well as Pandas, is creating a deletion route and keeping a running total of the tenant's balance. Given that the client wanted the ability to delete rows of data they inserted, I needed to first include a way to view the database and then incorporate a loop between the flask app and the database to handle deletions. This goes hand-in-hand with the tenant balance since deleting rows would affect this running tally that is being kept across records. So I spent a long time figuring out the logic of handling deletions and a running cumulative variable.
